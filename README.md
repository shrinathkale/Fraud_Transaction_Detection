# Fraud Transaction Detection System

A complete fraud detection system using Random Forest machine learning model trained on transaction data with MongoDB integration for customer data and OTP verification.

## Project Structure

```
.
├── fraud.csv                      # Raw transaction dataset
├── model.ipynb                    # Model training notebook
├── fraud_detection_model.pkl      # Trained model (generated by model.ipynb)
├── fraud_predict.py               # Fraud prediction script with DB integration
├── setup_db.py                    # Database setup script
├── requirements.txt               # Python dependencies
├── example_transactions.csv       # Example transactions for batch testing
└── README.md                      # This file
```

## Setup

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Setup MongoDB

Ensure MongoDB is running on `mongodb://localhost:27017/`.

Run the database setup script to insert sample customer data:

```bash
python setup_db.py
```

### 3. Train the Model (One-time)

Run the Jupyter notebook to train and save the model:

```bash
jupyter notebook model.ipynb
```

This will create `fraud_detection_model.pkl` in the current directory.

**Important**: The notebook must complete successfully and save the model before running predictions.

## Feature Explanation

The model uses 9 features computed from transaction data:

| Feature | Description |
|---------|-------------|
| `amount` | Transaction amount |
| `oldbalanceOrg` | Sender's balance before transaction |
| `newbalanceOrig` | Sender's balance after transaction |
| `oldbalanceDest` | Receiver's balance before transaction |
| `newbalanceDest` | Receiver's balance after transaction |
| `errorBalanceOrig` | Balance discrepancy at origin account |
| `errorBalanceDest` | Balance discrepancy at destination account |
| `errorBalanceOrig_1` | Binary flag: error in origin (0/1) |
| `errorBalanceDest_1` | Binary flag: error in destination (0/1) |

### Error Balance Calculation

```
errorBalanceOrig = oldbalanceOrg - amount - newbalanceOrig
errorBalanceDest = oldbalanceDest + amount - newbalanceDest
```

If error = 0, the balance change is consistent. Non-zero errors may indicate fraud.

## Model Details

- **Algorithm**: Random Forest Classifier
- **Training Data**: `fraud.csv` (TRANSFER and CASH_OUT transactions only)
- **Train/Test Split**: 80/20 with stratification
- **Performance**: Optimized for fraud detection with class balancing

## Input Requirements

When using the prediction script, provide:

1. **customer_id**: Transaction identifier (string, any format)
2. **amount**: Transaction amount (positive number)
3. **oldbalanceOrg**: Sender's account balance before transaction (number ≥ 0)
4. **newbalanceOrig**: Sender's account balance after transaction (number ≥ 0)
5. **oldbalanceDest**: Receiver's account balance before transaction (number ≥ 0)
6. **newbalanceDest**: Receiver's account balance after transaction (number ≥ 0)

**Note**: The script validates input types but not business logic (e.g., negative balances). Ensure input values are realistic.

## Interpretation

- **Prediction = 0**: Transaction classified as NORMAL ✅
- **Prediction = 1**: Transaction classified as FRAUD ⚠️
- **Fraud Probability**: Confidence level (0% = definitely normal, 100% = definitely fraud)

### Decision Threshold

Default threshold is 50% (if probability > 0.5, classify as fraud). You can adjust this in the code:

```python
threshold = 0.5
is_fraud = probability > threshold
```

## Troubleshooting

### Error: "fraud_detection_model.pkl not found"

**Solution**: Run `model.ipynb` to train and save the model first.

### Error: "Invalid input. Please enter a valid number"

**Solution**: Ensure you enter numeric values (integers or decimals) only.

### All transactions predicted as normal/fraud

**Possible causes**:
- Input features not matching training data distribution
- Model needs retraining with updated data
- Threshold may need adjustment

## Advanced Usage

### Batch Predictions

Modify `fraud_predict.py` to read from CSV:

```python
import pandas as pd

df = pd.read_csv('transactions.csv')
results = []

for _, row in df.iterrows():
    features = compute_features(row['amount'], row['oldbalanceOrg'], ...)
    pred, prob, _ = predict_fraud(model, features)
    results.append({
        'customer_id': row['customer_id'],
        'prediction': pred,
        'probability': prob
    })

results_df = pd.DataFrame(results)
results_df.to_csv('predictions.csv', index=False)
```
